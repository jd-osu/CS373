Week 3 Write-Up

Defending against malware requires an understanding of the main stages of malware operations, attack vectors and attack flow. These include first contact, local execution, establishing a presence, and performing malicious activity.

First contact - Whatever delivery mechanism is being used by the attacker interacts with the host's machine. There are many possibilities for first contact, including email, instant messaging, malvertising (e.g. representing malware as legitimate software and inducing users to download it), poisoned search results, or what's known as a "watering hole," which is a term for sites that are regularly accessed by a user (which the user wouldn't otherwise suspect).

Local execution - Somehow inducing the user to run the program (or setting it up so that the program is run automatically.

Establish a presence - Here, the malware tries to blend in or hide on the host machine. In many cases, malware generates filenames, paths, or process names that are similar to legitimate processes of the operating system (for example). Another method is to change creation or modification timestamps to hide recent activity or newly created files.

Malicious activity - Once the malware is safely established on the host, it sets about performing whatever malicious activity it was intended for. Examples include harvesting information and/or contacting the attacker ("phoning home").

The different stages of an attack present different opportunities for defense. For example, at the innermost level (with respect to the end node or host device), access control determines which processes have access to which data or other privileges on the machine. Other defense mechanisms include anti-malware, host IPS, host firewall, message reputation, network reputation, web reputation, network firewall, and network intrusion prevention.

In general, a varied approach that layers multiple different defenses at different possible stages in the attack is very effective. In any malware defense system, the key is to provide enough functionality to allow content to do what it was designed to do while still providing strong protection.

One useful tool is Yara, which allows scanning files or memory based on a rule or signature representing a query of strings with logical conditions. YaraEditor provides an IDE for developing yara rules and testing the rules on sample data. In one method, rules are developed using YaraEditor and tested against the Windows32 folder to ensure that draft rules are not too generic that they will match with common or safe programs.

Fig. 1 - A sample UI screen for YaraEditor. Note that the string was not ultimately useful in defending against malware because the string matched with files in the Windows32 folder, which are necessarily safe.

Development of Yara signatures can also be automated, offering greater potential scale, precision, and consistency. On the other hand, automation is done out of context and is easy for attackers to evade or attack with probes or DoS attacks.

Cuckoo is one tool that performs automated analysis on malware. It runs on a host machine and controls one or more virtual machines and/or a virtual network. Cuckoo traces Windows API calls, files being created, deleted and/or downloaded, memory dumps of the malware processes, network traffic, screenshots of windows desktops taken during execution of the malware, and full memory dumps of the machines.

Tricky malware takes control of your browser and your computer

by Jason DiMedio

If you've ever noticed the home page or start page of your browser change all by itself, you could be infected with malware that directs the host to a malicious web page whenever the user starts up their browser.

Malware Hash: 00670F2B9631D0F97C7CFC6C764DD9D9

By modifying system settings, this piece of malware directs victims to a custom-generated URL that includes an address (shown to the left) along with a path consisting of some arbitrary numbers "999" and the current date. Whenever the browser is opened, this page will be accessed.

Malware often needs to communicate back to an attacker, whether to send status information, download more files or offload data extracted from the host machine.
