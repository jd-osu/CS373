The current objective is to analyze data files stored in a textual CSV format (e.g. .csv files), the files containing header data for packets exchanged across a particular network. The two different files are R.csv and O.csv, referred to as the R data set and the O data set respectively.

Ultimately, the goal is to determine the functions of the networks represented by each of the R and O data sets. For example, are these networks work, home, data center, or ISP networks?

A basic script ("scancsv.py") scans the .csv files and provides information such as the total number of bytes, the total number of packets, and the number of different protocols used. This script will be adapted to perform additional functionality throughout the process of analyzing the data sets.

  • 139- NetBIOS Session Service
  • 80- HTTP
  • 110- POP3

Protocol 139 for "NetBIOS Session Service" is often used for applications and computers to communicate over a local area network. Productivity software that operates on a network might use this protocol to send and receive application data.

HTTP is the protocol used for requesting and receiving web pages.

POP3 is an email protocol that is used by client programs to receive mail (e.g. as opposed email servers).

Thus, users of this network are apparently using email exchanging application data, accessing email via client programs and accessing web pages.

Based on this analysis it seems reasonable to assume that the R network is a work network such as one that would be set up at a corporate office used by workers working at the office.

Running the -stats script on the O data set produces the following output:

Analysis of the results with respect to the O data set shows a relatively high proportion of the traffic is destined for the following ports:

  • 25- SMTP
  • 80- HTTP
  • 22 - SSH (secure logins, file transfers, port forwarding)
  • 53- DNS
  • 445- Microsoft-DS

SMTP is an email protocol that is used to exchange mail between email servers and possibly for users to send email.

HTTP is the protocol used for requesting and receiving web pages.

SSH is used for remotely accessing servers (e.g. for file transfer) using an encrypted connection.

DNS is used to translate domain names and addresses to IP addresses.

Microsoft-DS (445) is a directory service offered by servers within Windows networks.

Thus, this network seems to be offering several higher level services that would typically be offered by servers (as opposed to clients operated by users).

Based on this analysis it seems reasonable to assume that the O network is for a data center, where large servers containing data (such as web applications, cloud-hosted enterprise data, and email) are housed and accessed via the internet.
